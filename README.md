# LOLGBPS

英雄聯盟 Ban/Pick 系列賽模擬與分析工具。

## 線上使用

部署後直接開啟網址即可，資料會同步到伺服器資料庫，同一連結的裝置會共用系列賽紀錄。

## 本機開發

```bash
npm install
npm run dev
```

- 前端：http://localhost:5173
- API：http://localhost:3001

本機若未設定 `DATABASE_URL`，API 會暫存於記憶體；瀏覽器仍會保留 localStorage 備份。

## 模擬器

1. 輸入藍方、紅方隊名
2. 勾選「我方」方位（藍左 / 紅右，每局可改，系列結束後鎖定）
3. 選擇 BO 場數，按「開始選角」
4. 從英雄列表拖曳或點選完成 Ban/Pick
5. 選角結束後按「藍方勝利」或「紅方勝利」
6. 若未勾選我方就宣告勝利，會提示先選擇我方

系列進行中可調整比分與局數；系列結束後紀錄會封存到系列賽紀錄。

## 系列賽紀錄

- 展開每場系列可檢視各局 Ban/Pick
- 點英雄頭像可標記路線（上野中下輔）
- 可填備註、時間軸事件（`@英雄名` 會顯示圖示）
- 路線未標滿的局不納入分析

## 分析

頂部篩選：日期、對手、勝敗、方位、僅路線完整局。

| 分頁 | 內容 |
|------|------|
| 總覽 | 場數、勝敗、路線分布 |
| 我方 | 英雄/路線/BAN 勝率、藍紅方位拆分 |
| 對手 | 依對手隊名分組；可篩選單一對手 |
| 隊伍 | 指定隊伍的整體數據 |

## Railway 部署

1. Fork 或 clone 此 repo
2. 在 [Railway](https://railway.app) 新建專案，連接 GitHub repo
3. 新增 **PostgreSQL** 服務，Railway 會自動注入 `DATABASE_URL`
4. 部署完成後開啟產生的網域

健康檢查：`GET /api/health`
